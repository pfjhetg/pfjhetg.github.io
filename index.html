什么是序列化

序列化就是将数据结构或对象状态转换成可供Unity保存和随后重建的自动化处理过程

Unity编辑器在很多时候会自动触发序列化：

  • 文件的保存/读取，包括Scene、Asset、AssetBundle，以及自定义的ScriptableObject等。
  • Inspector窗口
  • 编辑器重加载脚本脚本
  • Prefab和Instantiation

序列化规则

序列化规则简单来说有两点，一是类型规则，系统据此判断能不能对对象进行序列化；二是字段规则，系统据此判断该不该对对象进行序列化。当对象同时满足类型规则和字段规则时，系统就会对该对象进行序列化。

类型规则

能序列化的类型 不能序列化的类型
c#原生数据类型（int/string/enum…） 抽象类
Unity内置数据类型（Vector/Rect/Color…） 静态类
继承自UnityEngine.Object的类 泛型类
标记了[Serializable]属性的类 没有标记[Serializable]属性的类
Array，List 其他容器

字段规则

该序列化的字段 不改序列化的字段
public成员 private/const/readonly/static成员
标记了[SerializeField]属性的成员 标记 了[NonSerialized]属性的成员

编辑器退出前会对所有打开的窗口进行序列化并保存序列化数据到硬盘。在重启编辑器后，序列化系统读入序列化数据，重新生成对应的窗口对象。进行序列化时，只有满足序列化规则的对象的值得以保存，不满足规则的对象则被序列化系统忽略。

热重载(hot-reloading)

对Unity中的C#脚本进行修改可以即时编译，不需要重启编辑器就看看到效果，这是Unity编辑器的一个令人称赞的机制。你有没有想过它是怎么实现的呢？答案就是热重载。

当编辑器检测到代码环境发生变化(脚本被修改、点击播放)时，会对所有现存的编辑器窗口进行热重载序列化。等待环境恢复(编译完成、转换到播放状态)时，编辑器根据之前序列化的值对编辑器窗口进行恢复。

热重载序列化与标准序列化的不同点是，在进行热重载序列化时，字段规则被忽略，只要被处理对象满足类型规则，那么就对其进行序列化。

引擎对象的序列化

UnityEngine.Object及其派生类(比如MonoBehaviour和ScriptableObject)称为Unity引擎对象，它们属于引擎内部资源，在序列化时和其他普通类对象的处理机制上有着较大的区别。

引擎对象特有的序列化规则如下：

  • 引擎对象需要单独进行序列化。
  • 如果别的对象保存着引擎对象的引用，这个对象序列化时只会序列化引擎对象的引用，而不是引擎对象本身。
  • 引擎对象的类名必须和文件名完全一致。

于插件开发，我们最可能接触到的引擎对象就是ScriptableObject

自定义序列化

Unity为我们提供了ISerializationCallbackReceiver接口，允许我们在序列化前后对数据进行操作。它并不能让系统直接处理你的复杂数据结构，但它给了你机会让你把数据”加工”成为系统能支持的形式

内置类型的及其实现

计算机以比特序列存储数据，每个比特非0即1。通常计算机以2的整数次幂个比特作为块来处理内存，可寻址的最小内存块成为“字节（byte）”，存储的基本单元成为“字（word）”。计算机的字节一般是8个比特构成，32位机器的字是32个比特，64位机器的字是64个比特。

如何选择类型

  • 明确知道数值不为负数时候，使用无符号类型。
  • 浮点数使用double，float精度不够（6位），double精度10位；并且他们之间的开销相差无几，某些计算机double运算效率更高。

赋值给无符号类型一个超出表示范围的值，结果是初始值对无符号类型表示数值综合取模后的余数，反之结果则是未定义，可能会出错崩溃。

初始化

建议初始化每一个内置类型变量，防止引用未初始化变量引发的运行时故障。

const限定符

const必须定义时候初始化。

const的作用域是本文件，多个文件出现同名的const定义的变量实际上是分别独立的。如果希望共享，在其他文件中使用extern关键之声明一下，同时定义的时候也加上extern关键字（定义的时候加上extern表明这个const在其他地方会被共享）。

const引用称之为对常量的引用,通常被叫做常量引用。非const引用无法引用一个const的对象。const指针和const引用类似。

const引用一个非const对象，可以保证外部无法修改const的值，只能通过非const对象原始变量修改。

C# 泛型约束

Posted on 2019-02-21 | In C#

在泛型类型定义之后（类中或者方法中）使用 where 约束泛型类型，使用方式类似继承的语法。

可以直接参考微软的类型参数的约束（C#编程指南）

约束 说明
where T: struct 类型参数必须是值类型。 可以指定除 Nullable 以外的任何值类型。 有关更多信息，请参见使用可以为 null 的类型（C# 编程指南）。
where T : class 类型参数必须是引用类型；这一点也适用于任何类、接口、委托或数组类型。
where T : new() 类型参数必须具有无参数的公共构造函数。 当与其他约束一起使用时，new() 约束必须最后指定。
where T : <base class name> 类型参数必须是指定的基类或派生自指定的基类。
where T : <interface name> 类型参数必须是指定的接口或实现指定的接口。 可以指定多个接口约束。 约束接口也可以是泛型的。
where T : U 为 T 提供的类型参数必须是为 U 提供的参数或派生自为 U 提供的参数。

这几个可以混合使用，比如where T : class, new(); where T : class1, calss2;等
